# 论坛 MVP：信息架构 + 置顶帖/模板帖（草案）

> 目标：先把“论坛应该有什么板块、置顶帖要写什么、发帖模板长什么样”落到 docs，便于后续直接按图部署。

## 1) 板块信息架构（IA）

建议首发只做 6 个板块，避免过早复杂化：

1. 公告 / 规则（只读）
2. 新手上路（提问、安装、常见坑）
3. 项目进展（Roadmap、周报、里程碑）
4. 配额 / quota-proxy（使用、计费、TRIAL_KEY 申请、FAQ）
5. 开发者（API、SDK、贡献指南、Issue 讨论）
6. 作品展示 / 案例（用户分享、Demo、教程链接）

## 2) 置顶帖清单（每个板块 1-2 篇即可）

### 公告 / 规则
- [置顶] 论坛规则（友好、禁止内容、举报渠道）
- [置顶] 如何提问（必须包含：环境、复现步骤、日志片段、期望行为）

### 新手上路
- [置顶] 0 到 1：从注册到跑通第一个示例（官网教程链接）
- [置顶] 常见问题 FAQ（网络/Key/配额/报错）

### 项目进展
- [置顶] 项目路线图（按季度/按里程碑）
- [置顶] 每周进度汇总（链接到 docs/posts/weekly 或官网更新）

### 配额 / quota-proxy
- [置顶] TRIAL_KEY 获取与使用（手动发放流程 + curl 示例）
- [置顶] 配额与计费说明（usage/admin 输出字段解释）

### 开发者
- [置顶] 贡献指南（PR/commit 规范、如何本地跑）
- [置顶] API 变更记录（changelog）

### 作品展示 / 案例
- [置顶] 分享模板（截图/链接/成本/关键参数）
