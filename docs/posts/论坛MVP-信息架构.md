# 论坛 MVP：信息架构（IA）草案

目标：先用最少板块跑起来，支持「公告/新手/讨论/反馈」，后续再细分。

## 顶层导航
- 首页（最新/热门/精华）
- 新手入门（从零到能用）
- 配额与 quota-proxy（使用/常见问题）
- 开发者（部署/贡献/接口）
- 反馈与工单（Bug/需求/投诉）

## 板块（建议）
1. 公告（只读或强约束发帖）
   - 站点变更、版本发布、维护窗口、规则更新
2. 新手区
   - 仅允许提问与教程类内容；鼓励用模板发帖
3. 配额 / quota-proxy
   - TRIAL_KEY 申请、配额规则、限流与错误码
4. 开发与部署
   - docker compose、运维、监控、PR 规范
5. 反馈与建议
   - Bug 反馈、功能请求、体验吐槽（可匿名视引擎能力）

## 标签体系（先少后多）
- [新手] [配额] [quota-proxy] [部署] [故障] [建议] [已解决]

## 权限与运营（MVP）
- 管理员：发公告、置顶、封禁、编辑标签
- 普通用户：发帖、回复、编辑自己的帖（限时）
- 规则：
  - 先友好引导：重复问题 → 贴教程链接 + 标记已解决
  - 敏感内容/违法内容 → 直接移除 + 记录
