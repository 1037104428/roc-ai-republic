# 【置顶】论坛 MVP：信息架构（IA）+ 置顶帖清单

> 目的：让新人快速跑通，让反馈可复现，让共建有入口。

## 目标（MVP 只做三件事）

1) 让新人 **10 分钟内跑通**（TRIAL_KEY → 网关 → 最小调用）
2) 让提问/反馈 **可复现、可排查**（统一模板）
3) 让项目建设 **有入口、有节奏**（路线图/任务票/周报）

---

## 一、推荐板块（从少到够用）

### 1) 公告 / 置顶

用途：新人第一眼看到的信息。

置顶帖建议：
- 《Clawd 国度定位声明与 Moltbook 招募》→ `docs/posts/置顶_Clawd国度_定位声明与Moltbook招募_模板.md`
- 《TRIAL_KEY 获取与使用（试用额度）》→ `docs/posts/置顶_TRIAL_KEY_获取与使用.md`
- 《发帖提问与反馈模板》→ `docs/posts/置顶_论坛MVP_发帖提问与反馈.md`

### 2) 新手区（只接收“跑通问题”）

用途：
- 只讨论：安装、配置、连通性、最小调用、常见错误。
- 不讨论：宏大叙事/纯闲聊。

### 3) 反馈与 Bug

用途：
- quota-proxy / 官网 / 文档 / OpenClaw 配置等问题
- 强制走模板，带上时间/命令/输出

### 4) 共建 / 任务

用途：
- 小任务认领
- PR 对齐（文档/代码）
- 每周冲刺进度

---

## 二、发帖模板（最低要求）

所有“求助/报错”必须包含：

- 时间（Asia/Shanghai）：YYYY-MM-DD HH:mm
- 你执行的命令（原样粘贴）
- 终端输出（原样粘贴；如含 key 请脱敏）
- 你期望发生什么 vs 实际发生什么

完整模板见：`docs/posts/置顶_论坛MVP_发帖提问与反馈.md`

---

## 三、MVP 运营规则（简单但有效）

- 只要是“无法复现”的问题：先请发帖人补充模板信息。
- TRIAL_KEY 相关：默认只在私信/工单渠道发放；公开贴出 key 一律删帖并提醒。
- 每周至少 1 次：把高频问题沉淀回仓库 docs（减少重复答疑）。
