# Forum MVP 信息架构（第一版）

更新时间：2026-02-18（Asia/Shanghai）

## 1) 目标
- 让新用户 3 分钟内看懂「这里是什么、先看什么、怎么提问」。
- 让常见问题可被复用回答，减少重复沟通。
- 先跑通最小闭环：浏览 → 发帖 → 获得回答 → 沉淀到文档。

## 2) 顶层版块（MVP）
1. **公告与规则**（只读）
2. **新手起步**（教程、避坑、FAQ）
3. **问题求助**（按模板提问）
4. **经验分享**（案例、复盘、最佳实践）
5. **产品反馈**（bug/建议/路线图讨论）

## 3) 置顶帖（建议）
- 《先看这里：5 分钟完成第一次提问》
- 《提问模板（复制即用）》
- 《FAQ：安装、配置、常见报错》
- 《版本与变更：本周更新摘要》

## 4) 模板帖（草案）

### A. 求助帖模板
- 环境：OS / 版本 / 部署方式
- 目标：你想实现什么
- 已尝试：做过哪些步骤
- 报错：完整报错文本（不要截图）
- 最小复现：可选

### B. 反馈帖模板
- 问题类型：Bug / 建议 / 体验
- 期望行为：你希望系统如何表现
- 实际行为：现在发生了什么
- 影响范围：阻塞 / 高 / 中 / 低

### C. 经验帖模板
- 场景背景
- 方案步骤
- 结果与指标
- 踩坑与建议

## 5) 运营节奏（MVP）
- 每周 1 次「精选问题与答案」汇总。
- 每周 1 次「版本更新与已知问题」置顶更新。
- 对高频问题沉淀到 docs，并在帖子中回链。

## 6) 成功指标（前 30 天）
- 首响时间（提问到首次回答）
- 已解决率（有结论的帖子占比）
- 重复问题率（同类问题复发）
- 文档回链点击率
