# 论坛 MVP：信息架构 + 置顶/模板帖（草案）

> 目标：先把“发什么、怎么找、怎么问”固化下来，论坛引擎之后再换也不怕丢结构。

## 一、分区（建议）

1. 公告 / 新手必读
2. 新手问答（从 0 开始）
3. 资源发布（模型/数据/教程/工具）
4. 项目展示（Demo / 开源仓库 / 进度）
5. 招募与协作（找队友/悬赏/外包）
6. 反馈与建议（bug/体验/需求）

## 二、置顶帖（每区 1-2 个）

### 1) 全站置顶：《新手从 0 到 1（先看这篇）》
- 你是谁：完全小白 / 有一点基础 / 开发者
- 你想做什么：跑起来 / 部署 / 调接口 / 做产品
- 你需要什么：教程、key、额度、示例

### 2) 全站置顶：《提问模板（复制粘贴）》
```
【我在做什么】
【我卡在哪里】
【我已经尝试过】
【日志/报错】
【期望结果】
【环境】系统/浏览器/命令/版本
```

## 三、模板帖（可发帖时自动带出）

- 资源发布模板
- 项目展示模板
- 反馈/bug 模板

## 四、发帖规范（极简）

- 标题：一句话说明“做什么 + 卡在哪”
- 报错：贴完整命令 + 关键日志（不要只截一行）
- 复现：别人能照着 3 步跑出来
