# 论坛信息架构与置顶帖规划

## 论坛目标
- 为 Clawd 国度用户提供交流平台
- 集中管理 TRIAL_KEY 申请与问题求助
- 沉淀常见问题与解决方案

## 板块结构

### 1. 必读区（置顶）
- **新手入门（从 0 到 1）** - 注册、安装、接入全流程
- **TRIAL_KEY 申请与使用（复制粘贴版）** - 申请模板与使用指南
- **发帖模板与遇到问题怎么问才能最快解决（复制填写）** - 标准化提问格式

### 2. 核心功能区
- **TRIAL_KEY 申请** - 用户申请试用额度的专用板块
- **问题求助** - 使用过程中遇到的技术问题
- **功能建议** - 产品功能改进建议

### 3. 交流区
- **使用分享** - 用户使用经验、技巧分享
- **公告与更新** - 官方公告、版本更新通知
- **灌水闲聊** - 非技术交流

## 置顶帖模板

### 新手入门帖（ID: 1）
**标题：** 必读·新手入门（从 0 到 1）注册安装接入

**内容结构：**
1. 注册账号流程
2. 安装 OpenClaw（国内源优先）
3. 配置 TRIAL_KEY
4. 验证与首次使用
5. 常见问题排查

### TRIAL_KEY 申请帖（ID: 2）
**标题：** 必读·TRIAL_KEY 申请与使用（复制粘贴版）

**内容结构：**
1. 申请前准备
2. 发帖模板（复制填写）
3. 审核流程说明
4. 拿到 KEY 后的使用步骤
5. 额度说明与续期

### 发帖模板帖（ID: 4）
**标题：** 发帖模板与遇到问题怎么问才能最快解决（复制填写）

**内容结构：**
1. 问题求助模板
2. 错误信息如何提供
3. 环境信息收集指南
4. 如何描述复现步骤
5. 提问礼仪与最佳实践

## 部署要求
1. 使用 Discourse 或类似开源论坛系统
2. 支持 Markdown 格式
3. 支持用户注册与邮件验证
4. 支持管理员审核机制
5. 支持私信功能（用于 KEY 发放）

## 自动化脚本

### 置顶帖初始化脚本
项目提供了 `scripts/init-forum-sticky-posts.sh` 脚本，用于快速生成和部署标准化的置顶帖：

```bash
# 查看脚本帮助
./scripts/init-forum-sticky-posts.sh --help

# 生成模板文件供手动使用（默认）
./scripts/init-forum-sticky-posts.sh

# 自动创建置顶帖（需要API密钥）
FORUM_API_KEY="your-api-key-here" ./scripts/init-forum-sticky-posts.sh --auto
```

脚本功能：
1. 生成三个标准置顶帖的完整 Markdown 内容
2. 支持手动和自动两种部署方式
3. 提供论坛 API 集成（需要管理员权限）
4. 保存模板文件到 `./forum-templates/` 目录

### 其他相关脚本
- `scripts/check-forum-service.sh` - 检查论坛服务状态
- `scripts/quick-verify-forum.sh` - 快速验证论坛功能
- `scripts/check-forum-init.sh` - 检查论坛初始化状态
