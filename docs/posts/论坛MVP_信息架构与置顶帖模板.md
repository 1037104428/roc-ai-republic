# 论坛 MVP：信息架构（IA）+ 置顶帖/模板帖（草案）

目标：先把“版块怎么分、置顶帖怎么写、用户发帖怎么写”落地到仓库 docs/，再选论坛引擎部署。

## 1) 信息架构（建议最小版）

- 公告（只读）
  - 【置顶】新手必读：如何提问/如何汇报 Bug
  - 【置顶】行为准则 & 版规
  - 【置顶】更新日志（每周/每次发布）

- 新手区（最重要）
  - 新手报到 / 你在做什么
  - 一条龙教程反馈（看不懂就来这）

- 产品与服务
  - quota-proxy（配额/Key/使用量）
  - 官网（clawdrepublic.cn）
  - API（api.clawdrepublic.cn）

- 需求与路线图
  - 功能请求（Feature Request）
  - 投票/优先级讨论

- 问题与故障
  - 故障报告（Incident / Bug）
  - 已知问题（FAQ）

- 共建
  - 文档贡献
  - 代码贡献

## 2) 置顶帖：新手必读（模板）

标题：
- 【新手必读】如何提问 / 如何获得最快响应

内容（复制即可）：
1. 你在做什么：
2. 你期望发生什么：
3. 实际发生什么（截图/报错原文）：
4. 你运行的命令（粘贴）：
5. 你所在环境：OS/浏览器/时间（含时区）
6. 如果是 quota-proxy：贴一下 `curl .../healthz` 输出（脱敏）
